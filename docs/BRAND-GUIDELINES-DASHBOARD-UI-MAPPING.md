## 3. Color System

### Color Strategy

XamPreps should not rely on only two colors across the product.

The logo gives the brand its core identity:

- **Purple** = structure, intelligence, system
- **Orange** = energy, action, momentum

But the product UI needs a broader working color system so users can understand status, feedback, alerts, and progression faster.

So the brand should use:

- **Primary brand colors** for identity
- **Secondary functional colors** for product states and dashboard clarity

---

### A. Primary Brand Colors

| Color | Usage | Meaning |
| --- | --- | --- |
| Purple | Main UI, navigation, structure, branded surfaces | Intelligence, structure, trusted system |
| Orange | CTA buttons, highlights, emphasis, key progress moments | Energy, movement, motivation |

### Suggested Primary Palette

- **Primary Purple:** #6A0DAD
- **Accent Orange:** #F7931E
- **Light Purple:** #EDE3F8
- **Soft Orange Tint:** #FDE7CC

---

### B. Secondary Functional Colors

These colors should support usability, not compete with the brand.

| Color | Usage | Meaning |
| --- | --- | --- |
| Green | Correct answers, completed states, positive outcomes | Accuracy, success, completion |
| Blue | Informational cards, neutral data blocks, guidance areas | Clarity, trust, calm support |
| Red | Wrong answers, destructive actions, critical alerts | Error, caution, stop |
| Amber/Orange | Warning, caution, pending attention | Attention, review needed |

### Suggested Functional Palette

- **Success Green:** #16A34A
- **Success Green Light:** #DCFCE7
- **Info Blue:** #2563EB
- **Info Blue Light:** #DBEAFE
- **Error Red:** #DC2626
- **Error Red Light:** #FEE2E2
- **Warning Amber:** #F59E0B
- **Warning Amber Light:** #FEF3C7

---

### C. Color Role Rules

### Purple

Use purple for:

- navigation
- headers
- section titles
- dashboard structure
- selected states
- premium or core branded surfaces

Purple remains the dominant brand color.

### Orange

Use orange for:

- primary CTAs
- active progress moments
- attention points
- “start now”, “continue”, “review now” actions

Orange should signal momentum and action.

### Green

Use green for:

- correct responses in exams
- completion confirmations
- positive score movement
- finished tasks
- success banners

Green should carry the clearest “you got it right” meaning.

### Blue

Use blue for:

- informational cards
- guidance boxes
- dashboard support cards
- tips, insights, explanations, and neutral summaries
- parent/school summary modules where a calm, readable tone is better than a high-energy tone

Blue helps break visual monotony and improves dashboard scanability.

### Red

Use red for:

- wrong answers in exam flow
- validation errors
- destructive actions
- critical alerts
- caution states that need immediate correction

Red should be used carefully and only where real friction or risk exists.

---

### D. Important Clarification on Success States

Because purple is already dominant in the brand, it should still appear in:

- rewards
- badges
- XP systems
- achievement framing
- mastery states

But **green should represent correctness and direct success feedback**, especially inside the exam-taking flow.

This creates a useful split:

- **Green** = correct / completed / successful outcome
- **Purple** = growth / mastery / branded achievement system

Example:

- Correct answer message → green
- Badge earned card → purple with optional orange accent
- XP streak increase → purple card with orange or green micro-indicator

---

### E. Recommended Dashboard Card Distribution

To avoid overloading the interface with purple, use controlled color variety:

- **Purple cards** → core learning progress, navigation-linked summaries, brand moments
- **Blue cards** → tips, insights, guidance, neutral analytics
- **Green cards** → completed goals, strong performance, correct-answer summaries
- **Amber/orange cards** → pending reviews, reminders, action-needed prompts
- **Red states** → only for error, wrong response, or critical alerts

This gives the website and dashboards more depth without weakening the logo system.

---

### F. Usage Rules

- Purple stays dominant at brand level
- Functional colors support comprehension at product level
- Do not use all colors equally on one screen
- One screen should still feel primarily purple-led
- Green and red must stay consistent in exam feedback
- Blue should be the main neutral secondary color for cards and information blocks
- Red should never be used decoratively

---

### G. Accessibility Rule

Never rely on color alone.

Always pair color with:

- icons
- labels
- text states
- borders or fills

Example:

- correct = green + check icon + “Correct” label
- wrong = red + alert icon + “Incorrect” label
- info = blue + info icon

Everything should feel like:

- broken into steps
- structured into blocks
- easy to follow

### UI Elements Should Feel Like:

- cards
- blocks
- modules

- clutter
- long continuous text
- unclear flows

---

- angled shapes
- segmented lines
- step-based visuals

- overly playful shapes

---

- progress fills (left → right)
- step-by-step reveal
- segment lighting effect

This reinforces:

→ learning as progression

# PART 2 — DASHBOARD UI MAPPING

This is where the logo becomes the product experience.

---

## 1. Core UI Principle

**Every screen must answer:**

👉 “Where am I in my learning journey?”

---

## 2. Student Dashboard Mapping

### A. Progress Bar (Critical)

Design:

- segmented bar (like logo slices)
- each segment = learning stage

Example:

[ Practice ] [ Review ] [ Mastery ]

- grey = not started
- purple = in progress
- orange = completed

---

### B. XP & Streak Section

- XP = numeric progress
- Streak = consistency

Design:

- orange highlight when increasing
- subtle animation when updated

---

### C. Subject Cards

Each subject = one “segment”

Card shows:

- progress %
- last activity
- weak areas

Color logic:

- low progress → light purple
- high progress → deeper purple + orange indicator

---

### D. Weak Areas (Important)

This is where value is clear.

Design as:

- stacked segments
- each item = a “gap”

CTA:

→ “Review Now” (orange button)

---

## 3. Exam Experience Mapping

### A. Question Flow

Each question = one segment

- current question highlighted in orange
- completed = purple
- not attempted = grey

---

### B. Answer Feedback

Correct:

- highlight in orange
- quick positive feedback

Incorrect:

- neutral tone (not harsh)
- explanation emphasized

---

### C. AI Explanation

This is the “orange moment”

Design:

- distinct card
- slightly highlighted background

Purpose:

→ this is the breakthrough moment

---

## 4. Review (Spaced Repetition) Mapping

### Key Idea:

Learning loop = segments repeating

UI Design:

- circular or looping visual
- each revisit = reinforcement

States:

- due now → orange
- upcoming → purple

---

## 5. Parent Dashboard Mapping

### A. Child Overview

Each child = one progress system

Display:

- XP
- streak
- performance

Use:

- segmented bars per subject

---

### B. Risk Indicators

Show clearly:

- declining performance
- inactivity

Color:

- warning = soft orange (not red)

---

## 6. School Dashboard Mapping

### A. Cohort View

- aggregated segments
- average performance

Visual:

- stacked bars
- comparison between subjects

---

### B. Performance Distribution

- top performers
- mid
- struggling

Use segmented grouping

---

## 7. Admin Dashboard Mapping

### A. Content Health

- number of exams
- coverage by subject

Represent as:

- segmented completeness

---

### B. Platform Metrics

- users
- attempts
- engagement

Use:

- growth lines + segment indicators

---

## 8. Interaction Design Rules

Always follow:

- Action = orange
- Structure = purple
- Progress = segmented
