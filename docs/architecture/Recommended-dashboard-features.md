# Recommended dashboard features for XamPreps

**student dashboard = action**,

**teacher dashboard = intervention**,

**parent dashboard = visibility + nudges**,

**school dashboard = oversight + performance management**.

## **1) Student dashboard**

This should be the most active dashboard in the whole product.

### **Core features**

- **Today’s study plan**
    - questions due for review
    - next recommended subject
    - upcoming exam target
    - estimated study time for today
- **Continue where I stopped**
    - resume last paper
    - resume last mock
    - resume last review session
- **Exam practice modes**
    - topic practice
    - full past paper
    - timed mock exam
    - weak-area drill
    - quick 10-question sprint
- **Performance snapshot**
    - average score by subject
    - recent scores
    - accuracy trend
    - completion rate
    - time spent studying
    - streak / consistency
- **Weakness detection**
    - weakest subjects
    - weakest topics within subject
    - repeated mistakes
    - “questions you still miss often”
- **Review engine**
    - due for review today
    - missed yesterday
    - mastered topics
    - confidence level per topic
- **Feedback layer**
    - instant answer check
    - explanation after each question
    - model answer for open-ended questions
    - “why this is wrong” feedback
    - alternative accepted answers where needed

### **Motivation features**

- XP / points
- streaks
- badges
- weekly target
- class or friend leaderboard only if carefully controlled
- progress toward exam readiness

### **Smart features**

- personalized study recommendation
- “focus on this topic next”
- predicted readiness for upcoming exam
- revision schedule builder
- AI tutor / ask for explanation / simplify answer

### **Practical UX features**

- low-data mode
- mobile-first layout
- offline-friendly answer drafting where possible
- bilingual support if needed later
- bookmarks / save question for later
- report a bad question

### **Best-practice note**

Strong student dashboards do not just show stats. They always answer: **what should I do next right now?** That is why diagnostics, recommendations, memory-strength style indicators, and progress summaries keep appearing across IXL, Seneca, and Khan Academy.

## **2) Teacher dashboard**

This is the big gap in XamPreps today. Your own product material says teacher accounts are not yet confirmed as a role, so this is a major opportunity.

### **Core features**

- **Class overview**
    - number of linked students
    - active this week
    - average score by subject
    - completion rate
    - students at risk
- **Assignment tools**
    - assign a past paper
    - assign a topic quiz
    - assign timed mock
    - set deadline
    - set to one class or multiple classes
    - optional retake rules
- **Markbook / results**
    - assignment scores
    - completion status
    - first attempt vs latest attempt
    - time taken
    - improvement after retry
- **Student intervention view**
    - who is struggling
    - who is inactive
    - who is improving fast
    - topic-level difficulty clusters
    - common wrong answers
- **Content control**
    - recommend practice sets
    - share custom notes
    - pin revision resources
    - attach teacher comments to assignments
- **Exam-readiness tracking**
    - readiness by class
    - readiness by subject
    - predicted pass band
    - students needing urgent help

### **Communication features**

- class announcements
- reminder to complete assignment
- feedback notes to student
- notify parent for at-risk students
- message summary rather than free-form chat at first

### **Reporting features**

- export PDF/CSV
- print-friendly class report
- weekly class summary
- topic mastery heatmap
- ranking by improvement, not only raw marks

### **Best-practice note**

Teacher tools on Khan Academy, IXL, Quizizz, and Seneca all converge on the same things: assign work fast, see who did it, spot who needs help, and drill down from class level to student level.

## **3) Parent dashboard**

This should be simpler than teacher dashboard. Parents want confidence, not too much detail.

### **Core features**

- **Child overview**
    - linked children
    - current class / exam level
    - study streak
    - time studied this week
    - latest score
    - strongest and weakest subjects
- **Progress tracking**
    - score trend over time
    - recent papers taken
    - mock exam results
    - completion of assigned work
    - review consistency
- **Alerts**
    - child inactive for X days
    - score dropped
    - exam approaching
    - subscription expiring
    - weak subject needs attention
- **Support actions**
    - pay subscription
    - unlock paper
    - set study target
    - approve or view recommended study plan
    - receive weekly report
- **Simple interpretation**
    - “doing well”
    - “needs more practice in mathematics”
    - “strong improvement in English”
    - avoid heavy education jargon

### **Optional advanced features**

- compare this month vs last month
- see school-assigned tasks
- parent encouragement messages
- suggested home support tips

### **Best-practice note**

Khan Academy and IXL both treat the parent view as a linked account experience with child monitoring and family-friendly progress reporting, not as a full instructional console. That’s the right model for XamPreps too.

## **4) School dashboard**

This is for head teachers, directors of studies, academic coordinators, and school admins.

### **Core features**

- **School overview**
    - total students linked
    - total teachers linked
    - active users this week
    - average score by class
    - average score by subject
    - completion rates
    - mock exam participation
- **Performance breakdown**
    - by level
    - by class
    - by stream
    - by subject
    - by gender if school wants it later
    - by time period
- **Teacher performance / usage**
    - assignments created
    - classes active
    - student completion under each teacher
    - usage rate by teacher
    - intervention response rate
- **Risk management**
    - classes falling behind
    - subjects with weak outcomes
    - inactive cohorts
    - students at pass-risk
    - teachers or classes not using the platform
- **Reporting**
    - export school reports
    - weekly leadership summary
    - mock exam summary report
    - subject department reports
    - board-ready PDF summary
- **Administration**
    - invite teachers
    - assign teachers to classes
    - assign students to school
    - manage subscription or school license
    - set access window by term

### **Best-practice note**

The stronger school-level products don’t just show student marks. They show platform usage, implementation quality, teacher activity, and downloadable reporting. That pattern is clear in Quizizz, Khan Academy Districts, and Seneca school reporting.

## **Best-practice rules to adapt for XamPreps**

Do not copy every feature from big platforms. Use these rules instead:

1. **Each dashboard should answer one main question**
    - Student: what should I study now?
    - Teacher: who needs help now?
    - Parent: is my child on track?
    - School: where is performance weak?
2. **Do not overload parents**
    - They need summary, trend, alerts, and action.
    - They do not need raw item-level analytics by default.
3. **Teachers need intervention tools, not just charts**
    - A chart without “assign again,” “group these students,” or “message parents” is weak.
4. **Schools need exports**
    - If school leaders cannot download usable reports, the dashboard loses value.
5. **Student dashboard must feel alive**
    - Continue button
    - next recommended task
    - progress reward
    - due review
    - exam countdown
6. **Topic-level data matters more than overall averages**
    - “Math 58%” is weak.
    - “Fractions 42%, Geometry 71%, Word problems 35%” is useful.

## **My recommendation for XamPreps product direction**

For XamPreps, I would structure the product around these four dashboard promises:

- **Student:** Practice, learn, and improve every day
- **Teacher:** Assign work and spot problems early
- **Parent:** Track progress and support from home
- **School:** Monitor performance across classes and subjects

And I would add a **Teacher role as a first-class role**, not bury teachers under School. Your own documents already show that XamPreps has strong student/parent/school/admin foundations, but teacher is the missing operational layer between student activity and school oversight. That gap will matter a lot once schools start using the platform at scale.
